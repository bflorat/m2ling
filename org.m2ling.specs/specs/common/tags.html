• Most m2ling items have a tag
  • A tag group maps one or more tags, can be created from GUI using a syntax like : PARAM_TYPE{PARAM_DYN,PARAM_KEX} this creates a “PARAM_TYPE” tag group and automatically add the couple of new tags.
  • Search is case insensitive.
  • Search can mix tags and tag groups like : {PROJECT_1},foo. where {PROJECT} maps component1,component2
  • Most m2ling tools provide filters textfields used to filter by tag or tag groups (even diagrams)
  • A custom property can map a tag group (so user can select one of the tag for instance)
