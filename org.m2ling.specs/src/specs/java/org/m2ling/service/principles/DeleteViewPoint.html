Viewpoint update and deletion

Administrators or actors with adequate authorizations can update some parts of existing viewpoint or delete them.

Update

Very viewpoint attribute, including its name can be changed except its id.

Given these new settings :

  • id=id_vp1
  • id=vp1_new_name
  • id=desc2
  • id=comment2
  • id=tag3,tag4
  • id=SUGGESTED,VALIDATED,REFUSED
When we update it
, Then we the viewpoint should now have the following characteristics : comment=comment2 description=desc2 iconPath= id=id_vp1 name=vp1_new_name statusLiterals=SUGGESTED,VALIDATED,REFUSED tags=tag3,tag4.

A status can't be removed while it is used by at least one item.
Given a vp1 rule with status="VALIDATED" When we set VAL against vp1
Then the status literal update should FAIL.

If one of the status is removed from status literal, it should be removed from the available status.
Given a current status=SUGGESTED,VALIDATED,REFUSED
When its status literals is updated to SUGGESTED,VALIDATED,
Then the viewpoint should contain SUGGESTED,VALIDATED status literals.
