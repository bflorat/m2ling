    Notes

    • The abstraction level is relative and is selected by the end-user according to its needs in its viewpoint. For instance, it is possible to declare :
      • Component type (M2) : JEE application Server
      • Component (M1) : JBoss
      • Component instance (M0) : a running JBoss 5 instance
      • Component type (M2) : Application Server
      • Component (M1) : JEE AS
      • Component instance (M0) : a running JEE AS
      • Component type (M2) : JBoss 5 application Server
      • Component (M1) : Any JBoss 5.0.3 instance
      • Component instance (M0) : a running JBoss 5.0.3 instance
      • Component type (M2) : JBoss 5 application Server
      • Component (M1) : Any JBoss 5.0.3 instance

    Decisions

    • Component type's enumeration maps ArchitectureItem, not directly ComponentTYpe to allow setting both ComponentType and ComponentTypeGroup items. Others types should be rejected.
    • A CT and its possible bound CT can own totally independent references.
