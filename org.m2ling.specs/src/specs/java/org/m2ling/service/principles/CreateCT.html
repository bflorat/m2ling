Component type creation

Administrators or actors with adequate authorizations can create a new CT (component type) that will be used as type by the components created in the studio.

Nominal rules

A Component Type is uniquely identified by its id. Its name must be unique.

A new CT created with

  • id=id_vp1
  • id=id_ct11
  • id=ct11
  • id=description1
  • id=comment1
  • id=tag1,tag2
  • id=id_comp1,id_comp2
  • id=id_
should have following characteristics : comment=comment1 description=desc1 iconPath= id=id_vp123 name=vp123 statusLiterals=SUGGESTED,VALIDATED tags=tag1,tag2.

Bindings and instantiation factor

Instantiation factor enforces the number of possibles component instances for a component of a given type. A Component Type can't be reified if its instantiation factor is zero.
