Rules creation

Administrators or actors with adequate authorizations can create a new rule against a given viewpoint.

A ~ViewPoint~ Rule is uniquely identified by its id.

Example

Given a view point named id_vp1, a new rule created with :
  • id=rule_id2
  • name=rule2
  • description=desc2
  • comment=comment2
  • status=VALIDATED
  • priority=3
  • rationale=rationale2
  • exception=exception2
  • tags=tag1,tag2
should have following characteristics : comment=comment2 description=desc2 drop= exceptions=exception2 id=rule_id2 name=rule2 priority=3 rationale=rationale2 status=VALIDATED tags=tag1,tag2 vpID=id_vp1.

Controls

  • A null input should make service call FAIL
  • ID and name can't be null, void or void once trimmed
  • ID size can't be larger than 40 characters
  • Name and status literal size can't be larger that 200 characters
  • description and comment size can't be larger that 2000 characters
  • See Tags specifications for tags limits
