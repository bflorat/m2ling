Rules update

Administrators or actors with adequate authorizations can update rules.

Every rule attribute, including its name can be changed except its id.

Given these new settings :
  • id_rule=id_rule1
  • name=new_name
  • description=new description
  • comment=new comment
  • tags=new tag3,new tag4
  • status=SUGGESTED
  • priority=1
  • rationale=new_rationale
  • exceptions=new_exceptions
When we update the rule

Then the viewpoint should now have the following characteristics : comment=new comment description=new description drop= exceptions=new_exceptions id=id_rule1 name=new_name priority=1 rationale=new_rationale status=SUGGESTED tags=new tag3,new tag4 vpID=id_vp1.

Format controls

    A null input should make service call FAIL
    Given a rule , following service inputs should make the service :
    Case Id Name Description Rationale Exceptions Comment Tags Status Priority should :
    Nominal id_vp1 rule new_description new_rationale new_exceptions new_comment new_tags41,new_tag42 VALIDATED 4 PASS
    Unexisting status id_vp1 rule new_description new_rationale new_exceptions new_comment new_tags41,new_tag42 foo_status 4 FAIL with code INVALID_STATUS
