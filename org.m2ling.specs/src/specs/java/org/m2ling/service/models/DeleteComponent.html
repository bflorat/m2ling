Component deletion

#COMP20 : A component can't be removed while at least a single component instance of this type exists in the view.

In this case, the service should : FAIL with code COMP_EXISTING_INSTANCE
If the component has no associated component instance of its type, the deletion service should : PASS

#COMP21 : A component can't be removed while at least a single component is bound to it.

In this case, the service should : FAIL with code COMP_EXISTING_BINDING

#COMP22 : When a component without associated CI is dropped, any reference of it from another component should be dropped all together if the reference contains no more targets.

Given a component id_comp_vp1_v1_comp4.1
With a CONTAINS reference to the single id_comp_vp1_v1_comp5.1 target
When id_comp_vp1_v1_comp5.1 is dropped
Then the reference should be null.
Given a CT id_comp_vp1_v1_comp4.1
Without associated components
With a DEPENDS_ON reference to id_comp_vp1_v1_comp5.1 and id_comp_vp1_v1_comp5.2
When id_comp_vp1_v1_comp5.1 is dropped
Then the reference should be targets=[id=id_comp_vp1_v1_comp5.2 name=comp_vp1_v1_comp5.2 ] type=DEPENDS_ON.

#COMP23 : The component can't be dropped if at least a single component having reference to the component to be dropped owns associated component instance having at least a single reference to associated reference at the instance level.

Given a component id_comp_vp1_v1_comp4.3
With a CONTAINS reference to the single id_comp_vp1_v1_comp5.3 component
And its associated id_ci_vp1_v1_comp4.3.1 component instance
With a CONTAINS reference to id_comp_vp1_v1_comp5.3.1 component instance
When id_comp_vp1_v1_comp5.3 is dropped
Then the service should FAIL with code COMP_REF_IN_USE.
