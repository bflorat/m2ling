References rules

#COMP1 : Component references must follow types defined in their associated CT.

If a reference target has a wrong type
then the component creation should creation should : FAIL with code COMP_ILLEGAL_REFERENCE
If a reference target is right but has a wrong type (like RUNS instead of DEPENDS_ON)
then the component creation should creation should : FAIL with code COMP_ILLEGAL_REFERENCE

#COMP2 : A reference target can't be null.

In this case, the CT creation should : FAIL with code TARGET_NOT_FOUND

Binding rules

#COMP3 : If a CT is bound, all components of its type should be bound.

If a component has no bound component,
Then the component creation should : FAIL with code COMP_MISSING_BINDING

#COMP4 : If a component is bound to another component, the bound component type must be those defined in the first component type.

If the bound component type is wrong, the component creation should : FAIL with code COMP_ILLEGAL_BINDING

Note that rule previous rules ensure implided rules already implemented in associated component types. For instance, it is impossible to bind a component to an already-bound component (binding chain) but we don't need to test this because it is already check at the CT level.
