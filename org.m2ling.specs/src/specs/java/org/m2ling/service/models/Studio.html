Notes

  • For now, groups are not implemented
  • Views may or not have a project-grain. They can represent only a project sub-system or on the contrary shows a set of projects.
  • Every VP and views (with credentials filtering) are shown from the studio to facilitate cross-business and cross-project knowledges
  • Views are lazy-loaded from GUI to avoid memory use
  • It is not possible to drop a component, component instance or associated component type while bound from another VP.
  • If a component type is bound to another type, each component instances of one of its component must be bound to one of the bound type instances (if any) associated with the bound component.

Decisions

  • Component type's enumeration maps ArchitectureItem, not directly ComponentTYpe to allow setting both ComponentType and ComponentTypeGroup items. Others types should be rejected.
  • A CT and its possible bound CT can own totally independent references.
