Architecture

  • A layer should only be dependent upon next immediate layer

Code

  • Presentation
  • don't use composite beans if possible, use only primitive types to avoid using Vaadin nestedproperties
  • Controls must be done :
    • In presentation view as surface control interactively when possible and just before calling service layer. No need for advanced business controls here.
    • In service layer entry, we do extensive business controls. No need to controls DTO afterward.
    • Date are always UTC
  • Nullity
    • None presentation beans field is null
    • DTO field (including collections) can't be null
    • Domain objects can't be null thanks EMF (to be verified)
  • Tests
    • No UT for boilerplate code like beans, DTO converter. Test them through acceptance test with full range of fields.
